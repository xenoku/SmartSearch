# Распределенное хранилище данных с гибридным поиском документов

Система обеспечивает автоматический парсинг, распознавание языковых локалей, векторизацию и интеллектуальное ранжирование текстовой информации в изолированном защищенном контуре.

---

## 🌟 Ключевые особенности и научная новизна

Система реализует подход **гибридный поиск**, объединяя два принципиально разных метода поиска для достижения максимальной полноты и точности выдачи:

1. **Семантический контур (Нейросетевой)**: 
   * Извлечение эмбеддингов признаков (размерность 768) с помощью мультиязычной модели `paraphrase-multilingual` через локальный сервер **Ollama**.
   * Быстрый поиск ближайших соседей по косинусному расстоянию (`Cosine Distance`) с использованием графового векторного индекса **HNSW** (`pgvector`).
2. **Лексический контур (Классический)**:
   * Инвертированный полнотекстовый поиск (FTS) средствами **PostgreSQL**.
   * Динамический подбор морфологических словарей (`russian`, `english` и др.) на основе статистического анализа текста библиотекой `langdetect`.
   * Ускорение запросов за счет построения инвертированных GIN-индексов на базе иммутабельных PL/pgSQL функций.
3. **Алгоритм ранжирования RRF (Reciprocal Rank Fusion)**:
   * Результаты семантического и лексического контуров объединяются на стороне Python-бэкенда по математической формуле взаимного слияния рангов со сглаживающим коэффициентом \(k=60\). Это нивелирует «проклятие размерности» и гарантирует высокую релевантность документов.

## 🛠️ Технологический стек (Tech Stack)

* **Backend:** Python 3.11, FastAPI (асинхронное ядро ASGI), Pydantic v2 (строгая валидация данных), SQLAlchemy v2 (ORM).
* **Database & AI:** PostgreSQL 16, расширение `pgvector`, ИИ-платформа Ollama (`paraphrase-multilingual`).
* **ETL Pipeline (Парсеры):** `pypdf` (парсинг PDF), `python-docx` (парсинг документов Word), `beautifulsoup4` (очистка веб-артефактов).
* **Frontend:** Нативный стек HTML5, CSS3 (адаптивная Grid/Flex верстка), асинхронный JavaScript (Vanilla ES6, концепция Event Listeners без тяжелых фреймворков).
* **DevOps / IaC:** Docker, Docker Compose (контейнеризация и оркестрация инфраструктуры в один клик).

## 🔒 Безопасность и архитектурная изоляция

Проект спроектирован по корпоративным стандартам безопасности (Enterprise Security Best Practices):
* **100% Локальный ИИ:** Все вычисления и векторизация проходят внутри закрытого контура, конфиденциальные документы не отправляются на внешние сторонние API (Cloud AI Providers).
* **Сетевой экран Docker:** СУБД PostgreSQL (порт 5432) и Ollama (порт 11434) полностью изолированы внутри внутренней виртуальной сети контейнеров. Внешний доступ из операционной системы имеет **только бэкенд FastAPI на порту 8085**, что исключает прямые SQL-инъекции и несанкционированный доступ к весам моделей.

## ⚡ Быстрый запуск системы (Quick Start)

### Системные требования
* **ОЗУ (RAM):** минимум **4 ГБ** свободной оперативной памяти.
* **Свободное место:** не менее **4 ГБ** на жестком диске (SSD/HDD).
* **ПО:** Установленный и запущенный [Docker](https://docker.com).

### Инструкция по развертыванию «В один клик»

1. Клонируйте репозиторий на свой компьютер:
   ```bash
   git clone https://github.com/xenoku/SmartSearch
   cd smart_search
   ```
2. Запустите автоматический конвейер сборки:
   * **Windows:** Дважды кликните по файлу `start.bat` в проводнике.
   * **Linux / macOS:** Выполните в терминале команды:
     ```bash
     chmod +x start.sh
     ./start.sh
     ```
3. Скрипт автоматизации самостоятельно:
   * Проверит наличие и сгенерирует локальный файл настроек окружения `.env`.
   * Соберет Docker-образ бэкенда на базе `Dockerfile`.
   * Инициализирует СУБД PostgreSQL, накатит структуру таблиц, подключит векторные расширения и GIN-индексы.
   * Скачает и развернет веса нейросети из реестра Ollama (процесс отобразится в логах терминала).

## 🖥️ Использование и порты интерфейсов

После того как лоадер завершит скачивание модели, откройте браузер:

* **Клиентская зона поиска:** [http://localhost:8085/](http://localhost:8085/) — ввод лексико-семантических запросов (поддерживается нативная отправка по кнопке `Enter`), отображение интегрального индекса ВСР (RRF Score) и времени поиска в миллисекундах.
* **Административная панель:** [http://localhost:8085/admin](http://localhost:8085/admin) — управление репозиторием.
  * *Пароль по умолчанию:* `admin123` (задается в файле `.env`).
  * *Функция сидирования:* Кнопка генерации тестовой базы данных разворачивает батчами **10 005 структурированных документов** для демонстрации скорости приближенного поиска.
  * *Управление индексами:* Кнопки динамического включения и отключения HNSW-графа для наглядного сравнения Sequential Scan и Index Scan.
  * *Массовая загрузка:* Поддерживается импорт одиночных файлов и пакетный импорт директорий с документами с вашего ПК.
  * *Управление БД:* Вывод состояния БД с удобной навигацией по ее элементам и инструменты управления ими.
